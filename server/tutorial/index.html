Step 1

What is a Blockchain?

A blockchain is a linked list of blocks where each block contains a cryptographic hash of the previous block. This creates a tamper-evident chain — changing any block invalidates all subsequent blocks.

Block Structure

Each block contains:

Field Description
index Block number (0 for genesis, 1, 2, ...)
timestamp When the block was created
data Miner’s public key, email, and action
previous_hash Hash of the previous block
nonce Number found during mining (Proof of Work)
hash This block’s hash (depends on all fields above)
signature Miner’s digital signature

How Blocks Link Together

+----------------+     +----------------+     +----------------+
|  Block 0       |     |  Block 1       |     |  Block 2       |
|  (Genesis)     |     |                |     |                |
|                |     |                |     |                |
| prev: 000...   |     | prev: a4f...   |     | prev: 7c2...   |
| hash: a4f...   |---->| hash: 7c2...   |---->| hash: 0e1...   |
+----------------+     +----------------+     +----------------+

Key Properties

  • Immutability — Changing a block changes its hash, which breaks the chain link.
  • Transparency — Anyone can read the entire chain and verify every block.
  • Proof of Work — Mining requires computational effort, preventing spam.
💡 Hint

Our tutorial uses a single centralized server for simplicity. Real blockchains like Bitcoin use thousands of nodes that reach consensus. The cryptographic concepts are the same.

Step 2

SHA256 and Hashing

Cryptographic hash functions are the backbone of blockchain security. We use SHA256 for everything in this tutorial.

Properties of SHA256

  • Deterministic — Same input always produces the same output
  • One-way — Cannot reverse a hash to find the input
  • Fixed-size output — SHA256 always produces 256 bits (64 hex characters)
  • Avalanche effect — A tiny change in input completely changes the output

Try It Yourself

import hashlib

# Hash a simple string
h1 = hashlib.sha256(b"hello").hexdigest()
print(h1)  # 2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824

# Change one character — completely different hash
h2 = hashlib.sha256(b"Hello").hexdigest()
print(h2)  # 185f8db32271fe25f561a6fc938b2e264306ec304eda518007d1764826381969

Role in Blockchain

SHA256 is used in three critical ways:

  1. Key generation — email + random numbers hashed to create a deterministic seed
  2. Block hashing — All block fields combined and hashed
  3. Proof of Work — Search for nonce that makes hash start with zeros

Block Hash Format

block_string = f"{index}{timestamp}{json.dumps(data, sort_keys=True)}{previous_hash}{nonce}"
hash = hashlib.sha256(block_string.encode()).hexdigest()
💡 Hint

This exact format is used by both the server and your client code. When you implement block hashing, make sure your format string matches exactly — including sort_keys=True.

Step 3

ECDSA Key Pairs

Public-key cryptography lets you prove your identity without sharing a secret.

How It Works

  • Private key (SigningKey) — A 32-byte secret number. Used to sign data.
  • Public key (VerifyingKey) — Derived from the private key. Used to verify signatures.
  • ECDSA — Elliptic Curve Digital Signature Algorithm
  • secp256k1 — The specific curve used by Bitcoin (and this tutorial)

Key Generation Process

email + student_random + instructor_random
                  |
                  v
          +---------------+
          |   SHA256       |
          |   (32 bytes)   |
          +-------+-------+
                  | seed
                  v
          +---------------+
          |  SigningKey    |---- Private Key (keep secret!)
          |  (secp256k1)  |
          +-------+-------+
                  |
                  v
          +---------------+
          | VerifyingKey  |---- Public Key (share freely)
          +---------------+

Step 6

Proof of Work

Proof of Work requires finding a nonce that makes the block hash start with a certain number of zeros.

How Mining Works

nonce = 0     -> hash = "8a3f1b..."  (doesn't start with 0000)
nonce = 1     -> hash = "f29c04..."
nonce = 2     -> hash = "12ab9e..."
...
nonce = 47231 -> hash = "0000a8..."  FOUND IT!

With difficulty 4, the hash must start with 0000. This takes roughly ~65,536 tries on average, typically 0.5 to 5 seconds on modern hardware.

⚠ Warning

The server uses this exact same format to verify your block. If your format differs even slightly, your hash won’t match.

Digital signatures prove you mined the block without revealing your private key.

Signing Process

Block Hash (string)
      |
      v

| ECDSA Sign        |<---- Your Private Key
| (SHA256 + DER)    |
+---------+---------+
          |
          v
    Signature (hex)
Exercise: Part 5 — Signing

Sign Your Block

In assignment/part5_submit.py, implement:

sign_block(signing_key, block_hash):

  1. Sign block_hash.encode() using the signing key
  2. Use SHA256 as the hash function and DER encoding (sigencode_der)
  3. Return the signature as a hex string
💡 Hint
signature = signing_key.sign(
    block_hash.encode(),
    hashfunc=hashlib.sha256,
    sigencode=sigencode_der
)
return signature.hex()
Exercise: Part 5 — Submitting

Submit to the Server

submit_block(server_url, block):

  1. POST to {server_url}/mine with the block as the JSON body
  2. Return the server’s JSON response

Server Validation

The server validates your submission in this order:

  1. Block index matches expected next index
  2. Previous hash matches latest block’s hash
  3. Hash is valid (recomputed from fields)
  4. Proof of Work passes (leading zeros)
  5. Key derivation is correct (pubkey matches seed)
  6. Email has not already completed the assignment
  7. Action is "set_done"
  8. Valid ECDSA signature
Run the Final Submission
cd assignment
uv run python part5_submit.py

Expected Output

Server response: {'success': True, 'message': 'Block N added by you@ualberta.ca'}

Handling Errors

If your submission is rejected, check these common issues:

Error Solution
Invalid index Someone mined before you. Re-fetch /latest and rebuild your block.
Invalid previous_hash Chain has changed. Fetch the latest block again.
Invalid hash Your hash format string doesn’t match the server’s. Check sort_keys=True.
PoW failed Hash doesn’t start with enough zeros. Re-mine with correct difficulty.
Invalid signature Make sure you’re signing block_hash.encode() with DER encoding.
Already completed You already submitted successfully. You’re done!
✅ Congratulations!

If you see {'success': True, ...}, you have successfully mined and submitted a block to the blockchain. Your block is now permanently part of the chain and can be verified by anyone.
