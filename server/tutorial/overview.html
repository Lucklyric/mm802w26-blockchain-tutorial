Core Concepts

Four cryptographic primitives power this assignment. Each maps to a specific exercise.

Hashing

SHA-256

A one-way function that maps any input to a fixed 64-character hex string. Changing a single bit of input completely changes the output. Used for key seeds, block hashes, and Proof of Work.

Asymmetric Crypto

ECDSA Key Pairs

A private key (secret, used to sign) and a public key (shared, used to verify). Built on the secp256k1 elliptic curve — the same one Bitcoin uses. Your identity on the chain.

Consensus

Proof of Work

Finding a nonce that makes the block hash start with 0000. Requires ~65,000 guesses on average. This computational cost prevents spam and ensures only one miner wins at a time.

Authentication

Digital Signatures

Sign the block hash with your private key using ECDSA + DER encoding. The server verifies your signature with your public key — proving you mined the block, without revealing your secret.

Step-by-Step Flow

Each part builds on the previous one. Run them in order — the final script (Part 5) orchestrates everything.

P1

Generate Your Key Pair

part1_keygen.py

Your identity is derived deterministically: hash your email + random numbers with SHA-256 to produce a 32-byte seed, then use it as a secp256k1 private key.

email + randoms → SHA-256 seed → ECDSA keygen → private key + public key
email + student_random + instructor_random ↓ SHA-256 (32 bytes) ← deterministic seed ↓ SigningKey (secp256k1) ← private key (keep secret!) ↓ VerifyingKey ← public key (share freely) ↓ Saved to .keys file
P2

Explore the Blockchain

part2_explore.py

Read the current chain state via simple REST API calls. Understand what’s on the chain before you add to it.

GET /chain → all blocks
GET /status → length, difficulty, ledger
GET /latest → most recent block
GET /difficulty → target prefix "0000"
P3

Create a Block

part3_create_block.py

Build the block data structure by fetching the latest block and constructing the next one. Nonce and hash are placeholders at this stage.

latest block → build block dict → block (nonce=0, hash="")
{ index: latest.index + 1, timestamp: current time, data: { miner_pubkey: your public key, miner_email: your email, student_random: [your numbers], instructor_random: [class numbers], action: "set_done" }, previous_hash: latest.hash, ← links to chain tip nonce: 0, ← placeholder hash: "", ← placeholder signature: "" ← placeholder }
P4

Mine the Block (Proof of Work)

part4_mine.py

Brute-force nonces starting from 0. For each nonce, compute the SHA-256 hash of the block. Stop when the hash starts with 0000 (difficulty 4).

block + nonce=0 → SHA-256 loop (~65k tries) → block with valid hash
hash format: f"{index}{timestamp}{json.dumps(data, sort_keys=True)}{prev_hash}{nonce}" nonce = 0 → hash = "8a3f1b..." x no match nonce = 1 → hash = "f29c04..." x nonce = 2 → hash = "12ab9e..." x ... ~65,000 tries on average nonce = 47231 → hash = "0000a8..." FOUND! starts with "0000"
P5

Sign & Submit

part5_submit.py

Sign the mined block hash with your private key (ECDSA + DER encoding), attach the signature, and POST the complete block to the server.

block hash → ECDSA sign (private key) → signature (hex) → POST /mine → success!
block hash (string) ↓ +——————————————————+ | ECDSA Sign |←— your private key | (SHA256 + DER) | +———————+——————————+ ↓ signature (hex string) ↓ block["signature"] = sig ↓ POST /mine → server validates → [ok] added to chain!

Hint — Server Validation

The server independently re-derives your public key from email + student_random + instructor_random. This means you cannot use someone else’s key — the server will reject the mismatch. All 8 checks must pass for your block to be accepted.

Warning — Race Condition

The chain advances as other students mine. If you get an "Invalid index" or "Invalid previous_hash" error, it means someone mined before you. Just re-run Part 5 — it fetches the latest block and re-mines automatically.

Grading

Once your block is on the chain, you’re done. The instructor checks the ledger at /status — if your email shows status: 1, you receive full credit. Open the Chain Explorer to verify your block.
