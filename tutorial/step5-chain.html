Step 5: Reading the Chain

The blockchain server exposes several GET endpoints that let you inspect the current state. Understanding the chain is essential before constructing your block.

Available Endpoints

EndpointReturns
GET /chainThe full blockchain (list of all blocks)
GET /statusChain length, difficulty, latest hash, full ledger
GET /latestThe most recent block
GET /difficultyCurrent difficulty and target prefix

Exercise: Part 3

Open client/part3_explore.py and implement four simple GET functions:

get_chain(server_url)

GET /chain and return the JSON response

get_status(server_url)

GET /status and return the JSON response

get_latest_block(server_url)

GET /latest and return the JSON response

get_difficulty(server_url)

GET /difficulty and return the JSON response

Hint: Each function is a single requests.get() call followed by .json(). Very straightforward!

Verification

uv run python part3_explore.py

You should see:

  • The chain with at least the genesis block (Block 0)
  • Your email in the ledger with status 0
  • The current difficulty (should be 4)
  • The latest block's hash (you'll need this for Part 4)

What to Look For

Pay attention to these fields in the latest block — you'll need them when constructing your own:

  • index — Your block will be index + 1
  • hash — Your block's previous_hash will be this value
