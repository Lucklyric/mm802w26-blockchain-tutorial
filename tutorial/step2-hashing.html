Step 2: SHA256 and Hashing

Cryptographic hash functions are the backbone of blockchain security. In this tutorial, we use SHA256 for everything: key generation, block hashing, and Proof of Work.

What is a Hash Function?

  • Deterministic — Same input always produces the same output
  • One-way — You cannot reverse a hash to find the input
  • Fixed-size output — SHA256 always produces 256 bits (64 hex characters)
  • Avalanche effect — A tiny change in input completely changes the output

Try It Yourself

Open a Python terminal and experiment:

import hashlib

# Hash a simple string
h1 = hashlib.sha256(b"hello").hexdigest()
print(h1)
# 2cf24dba5fb0a30e26e83b2ac5b9e29e1b161e5c1fa7425e73043362938b9824

# Change one character — completely different hash
h2 = hashlib.sha256(b"Hello").hexdigest()
print(h2)
# 185f8db32271fe25f561a6fc938b2e264306ec304eda518007d1764826381969

# Hashes are always 64 hex characters (256 bits)
print(len(h1))  # 64

Role in Blockchain

SHA256 is used in three critical ways:

  1. Key generation — Your email + random numbers are hashed to create a deterministic seed
  2. Block hashing — All block fields are combined into a string and hashed
  3. Proof of Work — You search for a nonce that makes the hash start with zeros

Block Hash Format

In this tutorial, a block's hash is computed as:

block_string = f"{index}{timestamp}{json.dumps(data, sort_keys=True)}{previous_hash}{nonce}"
hash = hashlib.sha256(block_string.encode()).hexdigest()
Important: This exact format is used by both the server and your client code. When you implement block hashing in Part 5, make sure your format string matches exactly — including sort_keys=True in the JSON serialization.
