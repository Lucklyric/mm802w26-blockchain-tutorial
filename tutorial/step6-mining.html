Step 6: Proof of Work

This is the most important step. Proof of Work (PoW) requires you to find a nonce value that, when included in the block hash computation, produces a hash starting with a specific number of zeros.

How Mining Works

  nonce = 0     -> hash = "8a3f1b..."  (doesn't start with 0000)
  nonce = 1     -> hash = "f29c04..."
  nonce = 2     -> hash = "12ab9e..."
  ...
  nonce = 47231 -> hash = "0000a8..."  FOUND IT!

With difficulty 4, your hash must start with 0000. Since SHA256 outputs are essentially random, you need to try roughly 65,536 (16^4) nonces on average. This takes about 0.5 to 5 seconds.

Block Hash Computation

The hash is computed from a string with this exact format:

block_string = f"{index}{timestamp}{json.dumps(data, sort_keys=True)}{previous_hash}{nonce}"
hash = hashlib.sha256(block_string.encode()).hexdigest()
Critical: The server uses this exact same format to verify your block. If your format differs even slightly (extra spaces, different key ordering), your hash won't match and the server will reject your block. Use json.dumps(data, sort_keys=True) for deterministic serialization.

Exercise: Part 4 — Create Block Structure

Open client/part4_create_block.py and implement:

create_block(server_url, email, public_key_hex)

  1. Fetch the latest block from /latest
  2. Build a block dict with:
    • index: latest block's index + 1
    • timestamp: current time (time.time())
    • data: {"miner_pubkey": ..., "miner_email": ..., "action": "set_done"}
    • previous_hash: latest block's hash
    • nonce: 0 (placeholder)
    • hash: "" (placeholder)
    • signature: "" (placeholder)

Exercise: Part 5 — Mine the Block

Open client/part5_mine.py and implement:

compute_block_hash(block)

  1. Build the block string using the exact format above
  2. Return hashlib.sha256(block_string.encode()).hexdigest()

mine_block(block, difficulty)

  1. Start with nonce = 0
  2. Set block["nonce"] = nonce
  3. Compute hash with compute_block_hash(block)
  4. If hash starts with "0" * difficulty, set block["hash"] and return the block
  5. Otherwise increment nonce and repeat

Verification

uv run python part4_create_block.py  # Should show block structure
uv run python part5_mine.py          # Should mine in a few seconds

After mining, the block's hash should start with 0000.

Tip: If mining takes more than 30 seconds, double-check your hash format string. A mismatch means every nonce produces an "incorrect" hash.
