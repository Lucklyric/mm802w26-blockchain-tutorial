Step 1: What is a Blockchain?

A blockchain is a linked list of blocks where each block contains a cryptographic hash of the previous block. This creates a tamper-evident chain — changing any block invalidates all subsequent blocks.

Block Structure

Each block in our chain contains:

FieldDescription
indexBlock number (0 for genesis, 1, 2, ...)
timestampWhen the block was created
dataMiner's public key, email, and action
previous_hashHash of the previous block
nonceNumber found during mining (Proof of Work)
hashThis block's hash (depends on all fields above)
signatureMiner's digital signature

How Blocks Link Together

  +----------------+     +----------------+     +----------------+
  |  Block 0       |     |  Block 1       |     |  Block 2       |
  |  (Genesis)     |     |                |     |                |
  |                |     |                |     |                |
  | prev: 000...   |     | prev: a4f...   |     | prev: 7c2...   |
  | hash: a4f...   |---->| hash: 7c2...   |---->| hash: 0e1...   |
  +----------------+     +----------------+     +----------------+

Each block's previous_hash points to the hash of the block before it. The genesis block (Block 0) has a fixed previous hash of all zeros.

Key Properties

  • Immutability — Changing a block changes its hash, which breaks the chain link from the next block.
  • Transparency — Anyone can read the entire chain and verify every block.
  • Proof of Work — Mining a block requires computational effort, preventing spam.
Note: Our tutorial uses a single centralized server for simplicity. Real blockchains like Bitcoin use thousands of nodes that reach consensus. The cryptographic concepts are the same.

Your Goal

By the end of this tutorial, you will add your own block to the chain. This requires:

  1. Generating a unique cryptographic key pair
  2. Registering your public key with the server
  3. Constructing a valid block
  4. Mining it (finding a valid Proof of Work)
  5. Signing and submitting it

No coding in this step — just make sure you understand the structure before moving on.
