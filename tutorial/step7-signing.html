Step 7: Digital Signatures

Digital signatures prove that you mined the block, without revealing your private key. Only the holder of the private key can create a valid signature, but anyone with the public key can verify it.

Signing Process

  Block Hash (string)
          |
          v
  +-------------------+
  | ECDSA Sign         |<---- Your Private Key
  | (SHA256 + DER)     |
  +---------+---------+
            |
            v
      Signature (hex)

Why Sign the Hash?

We sign the block hash string rather than the raw block data. Since the hash already commits to all block fields (index, timestamp, data, nonce, etc.), signing the hash is equivalent to signing the entire block — but simpler to implement.

Exercise: Part 6 (Signing)

Open client/part6_submit.py and implement:

sign_block(signing_key, block_hash)

  1. Sign block_hash.encode() using your signing key
  2. Use SHA256 as the hash function
  3. Use DER encoding (sigencode_der)
  4. Return the signature as a hex string
Hint: The signing call looks like: 
signature = signing_key.sign(
    block_hash.encode(),
    hashfunc=hashlib.sha256,
    sigencode=sigencode_der
)
return signature.hex()

DER Encoding

DER (Distinguished Encoding Rules) is a standard binary format for ECDSA signatures. It encodes the two signature components (r, s) in a compact, unambiguous way. The server expects DER-encoded signatures.
