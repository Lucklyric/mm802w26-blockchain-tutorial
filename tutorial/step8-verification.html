Step 8: Submit and Verify

You've built the block, mined a valid hash, and signed it. Now submit it to the blockchain!

Server Validation

When you submit a block via POST /mine, the server checks (in order):

  1. Block index — Must be the next expected index (chain length)
  2. Previous hash — Must match the latest block's hash
  3. Hash validity — Recomputed hash must match the submitted hash
  4. Proof of Work — Hash must start with the required number of zeros
  5. Registration — Your public key must be registered
  6. Not already mined — Your status must be 0 (not already completed)
  7. Correct action — Must be "set_done"
  8. Valid signature — ECDSA signature must verify

Exercise: Part 6 (Submitting)

In client/part6_submit.py, implement:

submit_block(server_url, block)

  1. Send a POST request to {server_url}/mine
  2. The JSON body is the entire block dict
  3. Return the server's JSON response

Running Part 6

uv run python part6_submit.py

This script does everything: creates a block, mines it, signs it, and submits it. On success, you'll see:

Server response: {'success': True, 'message': 'Block N added by you@ualberta.ca'}
SUCCESS! Your block is on the chain. Status: DONE

Handling Errors

"Invalid index. Expected N"

Someone else mined a block while you were mining yours. The chain has grown, so your block's index and previous_hash are now stale. Simply re-run part6_submit.py — it will fetch the latest chain state and mine a new block.

This is how real blockchains work! Miners compete, and the first valid block wins. Losers must re-mine against the new chain tip. This is a feature, not a bug.

"Miner already completed"

You've already successfully mined a block. Check /status to confirm.

"Hash mismatch"

Your block hash computation doesn't match the server's. Double-check the format string in Part 5.

"Invalid signature"

Check that you're signing block_hash.encode() (the hash string, not the block data) with DER encoding.

Final Verification

Open your browser and visit:

http://<server-ip>/status

Find your email in the ledger. If status is 1, you've completed the assignment!

Congratulations! You've successfully:
  • Generated an ECDSA key pair from a deterministic seed
  • Registered your identity on a blockchain
  • Constructed and mined a block using Proof of Work
  • Signed and submitted your block with a digital signature
These are the core building blocks of Bitcoin and most other blockchains.
