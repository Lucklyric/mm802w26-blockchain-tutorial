Step 4: Registration

Before you can mine a block, you must register your public key with the blockchain server. This associates your email with your public key in the server's ledger.

The Ledger

The server maintains a ledger that tracks each registered miner:

FieldDescription
emailYour @ualberta.ca email (the key)
pubkeyYour public key hex string
status0 = registered, 1 = block mined
block_indexWhich block you mined (after mining)

REST API

Send a POST request to the server:

POST /register
Content-Type: application/json

{
    "pubkey": "your_public_key_hex",
    "email": "yourname@ualberta.ca"
}

Exercise: Part 2

Open client/part2_register.py and implement:

register(server_url, email, public_key_hex)

  1. Send a POST request to {server_url}/register
  2. Include JSON body with pubkey and email
  3. Return the server's JSON response
Hint: Use requests.post(url, json={"pubkey": ..., "email": ...}) and response.json().

Verification

uv run python part2_register.py

Expected output: {"success": true, "message": "Registered yourname@ualberta.ca"}

Note: You can only register once. If you try to register again, the server will return an error. This is expected — just proceed to the next step.
